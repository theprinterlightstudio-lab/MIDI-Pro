Troubleshooting

Installation

The plugin doesn't appear after import.
Restart Touch Portal completely after importing. If it still doesn't appear, confirm you're importing a .tpp file, not a renamed .zip.
I have an old pre-release version installed too.
Remove com.sfx.midipro (the old identity) from Settings → Plug-ins before installing this build — they're treated as separate plugins.

MIDI Devices

A MIDI In or MIDI Out dropdown is empty.
Make sure the device is connected (or your virtual port app is running) before opening the action's settings, then reopen it.
Values seem “off by one direction.”
For CC Relative / CC Comp, check which value your DAW expects for increase vs decrease. Use MIDI Learn on a working relative knob to see the exact byte pattern.

Ports

Nothing happens when I press a button at all.
Check that MIDI Out is actually set — an unset port silently does nothing rather than erroring, by design.

Permissions

Another application seems to be blocking my device.
Many MIDI devices only allow one exclusive connection at a time. Close other software (DAWs, other controller utilities) that might already have the port open.

Touch Portal

I named a control, pressed it, but nowhere shows the value.
Expected — naming only creates the state in the background. Add Update Button text… under some button's On Event tab and point it at that state yourself.
My Toggle button doesn't seem to alternate correctly.
Toggle mode ignores button release entirely — only presses count. Make sure you're releasing between presses.

Windows

Windows SmartScreen warns about the executable.
This is standard for smaller-distribution Windows binaries without an EV code-signing certificate. Click “More info” → “Run anyway” if you trust the source you downloaded it from.
