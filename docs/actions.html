MIDI Actions

MIDI Pro ships seven actions under the MIDI Pro category in Touch Portal's action list. There is no separate "Transport" send action — see the callout near the bottom of this page for what MIDI Pro actually offers for transport, which is a different (and passive) kind of feature.

Mode behavior — read this first (MIDI Note & CC share these six modes)

The names are not what you'd guess — "On/Off" is a one-shot pulse, "Momentary" is the true press-and-hold.

ModeBehavior
On / OffPress fires ON and OFF immediately, back to back — a one-shot pulse. Use for drum hits, one-shot samples.
On onlyPress sends only the ON velocity. Nothing on release.
Off onlyPress sends only the OFF velocity.
HoldPress sends ON. Release does nothing — turn it off some other way.
MomentaryPress sends ON, release sends OFF — a true held key, like a sustain pedal. What most people expect from “On/Off.”
ToggleEach press alternates ON → OFF → ON. Release is ignored. Good for mute buttons that stay lit.
The mode people actually want, most often "Hold the button down, it plays; let go, it stops" is Momentary, not "On / Off."

MIDI Note

MIDI Note NOTE ON / OFF
Sends a Note On / Note Off message — the message a piano key generates.
NameOptional. Fill in for live states/events — see Runtime Telemetry.
ModeOn/Off · On only · Off only · Hold · Momentary · Toggle — see above.
Note0–127. Which note. Middle C = 60.
Velocity On / Off0–127 each. How hard the on/off hit is.
Ch1–16. MIDI channel.
MIDI Out / InDestination port; In optionally mirrors incoming for Toggle feedback.

Best practice: use Momentary for anything that should stop when released (sustained notes), and On/Off for one-shot triggers (samples, drum hits).

Control Change (CC)

Control Change (CC) CONTROL CHANGE
The general-purpose knob message. Same shape as MIDI Note, for continuous parameters instead of notes.
NameSame as MIDI Note.
ModeIdentical six modes as MIDI Note.
CC#0–127. Which control.
Value On / Off0–127 values sent for the two states.
Ch1–16.
Repeat Mode / RateOnce, or Repeat While Held at 5–50 Hz — continuously re-sends Value On while the button is down.

Best practice: Repeat While Held is ideal for "nudge this fader up while I hold the button" workflows in DAWs that support incremental CC nudges.

CC Relative (CW/CCW)

CC Relative (CW/CCW) ENCODER BUTTON
A button-based rotary encoder. Two completely independent packets — one for clockwise, one for counter-clockwise.
NameNames the encoder — CW and CCW share one live position.
CW CC# / Val / ChThe exact CC sent when you press the button.
CCW CC# / Val / ChThe exact CC sent when you release it.
Repeat / RateOnce, or Repeat While Held — held repeats CW at Rate; release always fires CCW once.

Best practice: check your DAW's relative-CC mapping docs before picking values — Binary Offset's 65 (CW) / 63 (CCW) pair is the most universally supported starting point.

Pitch Bend

Pitch Bend PITCH WHEEL
Sends the 14-bit pitch wheel message.
NameOptional, for telemetry.
ModeFixed — press sends Value only. Return to Center — press sends Value, release snaps back to 8192.
Value0–16383. 8192 = center/no bend.
Ch1–16.
MIDI OutDestination port.

Best practice: use Return to Center for a spring-loaded pitch-bend feel from an ordinary button.

Program Change

Program Change PATCH SWITCH
Switches a synth or instrument's patch/preset number.
NameOptional, for telemetry.
ModeFixed sends the exact Program every press. Increment/Decrement step ±1 from wherever it last landed — does not reset.
Program0–127. Starting/fixed patch number.
Ch1–16.
RateRepeat speed if held on Increment/Decrement.

Best practice: pair one Increment and one Decrement action with the same Name to build a two-button patch browser — see Runtime Telemetry for how to display the current patch number.

Panic / All Notes Off

Panic / All Notes Off EMERGENCY STOP
Sends the MIDI "all notes off" broadcast so any stuck or hanging notes clear instantly.
ChannelAll, or a specific channel 1–16.
MIDI OutDestination port.

Best practice: put one on every page. You will use it more than you expect.

MIDI Learn

Not a sender — a diagnostic listener. See the dedicated MIDI Learn page.

A note on "Transport"

This is not a send action MIDI Pro has no button you press to control DAW transport. What it does have is passive transport feedback: 8 read-only states (state.transport.play/stop/record/pause/loop/click/punchin/punchout) that light up automatically when your DAW sends Mackie Control Universal (MCU) style transport LED messages to a MIDI input port MIDI Pro is listening on. If your DAW doesn't speak MCU on that port, these states simply never change — there's no action to trigger them yourself.
